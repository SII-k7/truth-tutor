# Truth Tutor 核心竞争力优化计划

## 目标
将 truth-tutor 从"更好的 prompt 工具"升级为"诊断学习平台"

## 优化方向

### 1. 增强学习档案系统 (learning-profile.mjs)
**目标**: 从简单记录升级为知识缺口分析

**新增功能**:
- [ ] 知识缺口模式识别（识别用户的学习弱点模式）
- [ ] 学习进度追踪（显示改进趋势）
- [ ] 改进建议生成（基于历史诊断）
- [ ] 学习效率指标（诊断→改进的时间）

**新增字段**:
```json
{
  "gapPatterns": {},        // 知识缺口的分类和模式
  "progressMetrics": {},    // 学习进度指标
  "improvementRate": 0,     // 改进率
  "learningStyle": "",      // 学习风格识别
  "nextRecommendation": ""  // 下一步建议
}
```

### 2. 改进 Drill 系统 (drills.mjs + drill-tracker.mjs)
**目标**: 从静态指导升级为自适应练习系统

**新增功能**:
- [ ] 自适应难度调整（根据完成情况调整）
- [ ] Drill 完成情况追踪（记录完成率和改进效果）
- [ ] 智能 drill 生成（根据诊断结果自动生成）
- [ ] Drill 效果分析（显示哪些 drill 最有效）

**新增字段**:
```json
{
  "drillId": "",
  "type": "",               // drill 类型
  "difficulty": 1-5,        // 难度等级
  "completionStatus": "",   // 完成状态
  "improvementEffect": 0,   // 改进效果
  "timestamp": ""
}
```

### 3. 添加诊断可视化仪表板
**目标**: 创建可视化的学习进度和诊断报告

**新增页面**:
- [ ] `/dashboard` - 学习仪表板
- [ ] `/knowledge-map` - 知识缺口地图
- [ ] `/progress` - 学习进度曲线
- [ ] `/history` - 诊断历史对比

**新增功能**:
- 知识缺口分布图
- 学习进度时间线
- 诊断结果对比
- 改进建议卡片

### 4. 实现个性化学习路径推荐
**目标**: 根据诊断结果推荐学习资源

**新增功能**:
- [ ] 学习路径生成（基于知识缺口）
- [ ] 资源推荐（论文、教材、视频）
- [ ] 学习顺序建议（优先级排序）
- [ ] 社区知识库集成

**新增模块**: `src/learning-path-recommender.mjs`

## 实现顺序

1. **第一阶段**: 增强学习档案系统
   - 添加知识缺口模式识别
   - 添加学习进度追踪
   - 添加改进建议生成

2. **第二阶段**: 改进 Drill 系统
   - 扩展 drill-tracker 功能
   - 添加自适应难度调整
   - 添加 drill 效果分析

3. **第三阶段**: 添加诊断可视化仪表板
   - 创建 dashboard 页面
   - 创建 knowledge-map 页面
   - 创建 progress 页面
   - 创建 history 页面

4. **第四阶段**: 实现个性化学习路径推荐
   - 创建学习路径推荐模块
   - 集成资源推荐
   - 添加社区知识库

## 预期成果

- ✅ 从"prompt 工具"升级为"诊断学习平台"
- ✅ 用户可以看到自己的学习进度和知识缺口
- ✅ 系统可以自动推荐学习资源和练习
- ✅ 用户可以追踪改进效果
- ✅ 建立学习社区和知识库
